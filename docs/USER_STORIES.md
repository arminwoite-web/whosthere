# User Stories

## Story 1: Nachrichten senden und empfangen
**Als** Benutzer
**Möchte ich** Textnachrichten in Echtzeit senden und empfangen
**Damit** ich mit anderen Benutzern kommunizieren kann

**Akzeptanzkriterien:**
- [ ] Ich kann eine Nachricht an einen anderen Benutzer senden.
- [ ] Ich empfange Nachrichten von anderen Benutzern in Echtzeit.
- [ ] Gesendete Nachrichten werden im Chatverlauf angezeigt.
- [ ] Empfangene Nachrichten werden im Chatverlauf angezeigt.

**Priorität:** Hoch

## Story 2: Ungelesene Nachrichten hervorheben
**Als** Benutzer
**Möchte ich** ungelesene Nachrichten deutlich erkennen können
**Damit** ich keine wichtigen Informationen verpasse

**Akzeptanzkriterien:**
- [ ] Ungelesene Nachrichten werden fettgedruckt dargestellt.
- [ ] Sobald ich eine ungelesene Nachricht gelesen habe, ist sie nicht mehr fettgedruckt.
- [ ] Die Hervorhebung erfolgt sowohl in der Chat-Übersicht als auch im jeweiligen Chat-Fenster.

**Priorität:** Hoch

## Story 3: Initiales Projekt-Setup
**Als** Entwickler
**Möchte ich** ein grundlegendes Projekt-Setup für die Chat-Anwendung haben
**Damit** ich schnell mit der Entwicklung beginnen kann

**Akzeptanzkriterien:**
- [ ] Es existiert ein Projektverzeichnis mit einer grundlegenden Ordnerstruktur (z.B. für Frontend, Backend, Konfiguration).
- [ ] Es sind initiale Konfigurationsdateien vorhanden (z.B. für Datenbankverbindung, Server-Ports).
- [ ] Die grundlegenden Abhängigkeiten für Frontend und Backend sind definiert (z.B. `package.json`, `requirements.txt`).
- [ ] Das Projekt lässt sich über einen einfachen Befehl (z.B. `npm install && npm start`) starten.

**Priorität:** Hoch
